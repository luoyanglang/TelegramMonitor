# 宝塔面板部署

宝塔面板是国内最流行的服务器管理面板，本教程手把手教你部署。

## 前提条件

- 已安装宝塔面板
- 已安装 Docker 管理器（在宝塔软件商店安装）
- 已准备好配置信息（[获取教程](../config/bot-token.md)）

## 部署步骤

### 1. 安装 Docker 管理器

如果还没安装，进入 **软件商店** → 搜索 **Docker管理器** → 安装。

<!-- TODO: 截图 - 宝塔软件商店 Docker 管理器 -->

### 2. 进入 Docker 管理

点击左侧菜单 **Docker** 进入管理界面。

<!-- TODO: 截图 - 宝塔 Docker 菜单 -->

### 3. 拉取镜像

点击 **镜像** → **拉取镜像**，输入：

```
luoyanglangge/telegram-monitor:latest
```

点击拉取，等待完成。

<!-- TODO: 截图 - 拉取镜像 -->

### 4. 创建容器

点击 **容器** → **创建容器**。

<!-- TODO: 截图 - 创建容器按钮 -->

### 5. 填写容器配置

<!-- TODO: 截图 - 创建容器表单 -->

**基本设置：**

| 字段 | 填写内容 |
|------|----------|
| 容器名称 | `telegram-monitor` |
| 镜像 | 选择刚才拉取的镜像 |
| 重启策略 | `always` |

**环境变量：**

点击添加环境变量，依次添加：

```
BOT_TOKEN=你的Token
TELEGRAM_API_ID=你的API_ID
TELEGRAM_API_HASH=你的API_Hash
AUTHORIZED_USER_ID=你的用户ID
```

<!-- TODO: 截图 - 环境变量配置 -->

**目录映射（可选但推荐）：**

| 服务器目录 | 容器目录 |
|------------|----------|
| `/www/telegram-monitor/data` | `/app/data` |
| `/www/telegram-monitor/sessions` | `/app/sessions` |

### 6. 提交创建

点击 **提交** 按钮，容器会自动启动。

### 7. 查看状态

在容器列表中可以看到运行状态，点击 **日志** 查看运行日志。

<!-- TODO: 截图 - 容器列表和日志 -->

## 开始使用

部署成功后，在 Telegram 中找到你的 Bot，发送 `/start` 开始使用。

## 常见问题

### 镜像拉取很慢

配置 Docker 镜像加速：
1. 进入 Docker 管理器设置
2. 添加镜像加速地址

### 容器无法启动

1. 检查环境变量是否正确
2. 查看容器日志排查错误
